## 🛡️ Принципы безопасности и архитектуры
- ✅ **Полный запрет `any`**: Строгий режим TS, `unknown` + type guards, дженерики, branded types
- ✅ **Защита от XSS/CSRF**: DOMPurify, строгие CSP-заголовки, SameSite cookies, CSRF-токены, HTTP-only флаги
- ✅ **Предотвращение инъекций**: Параметризованные запросы, ORM (TypeORM/Prisma/Django ORM), строгая санитизация ввода
- ✅ **Mitigation DoS/ReDoS**: Rate limiting, лимиты глубины запросов, проверка сложности регулярок, таймауты выполнения
- ✅ **Защита от Path Traversal**: Нормализация через `path.resolve()`, allowlist-маршруты, виртуальная маршрутизация файлов
- ✅ **Безопасная работа с буфером/памятью**: Потоковая обработка, чанковая загрузка, контроль выделения ресурсов
- ✅ **Zero-Trust валидация**: Валидация схем на всех границах API, fail-fast обработка ошибок, структурированное логирование
